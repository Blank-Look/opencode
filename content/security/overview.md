# Security Overview

Security policies, controls, and procedures to protect ICT assets, data, and services from threats and vulnerabilities.

## Purpose

- Establish a security baseline across all systems
- Define access control and identity management standards
- Ensure compliance with regulatory requirements
- Provide incident response and threat management frameworks

## Security Domains

| Domain | Description |
|---|---|
| Threat Management | Detection, analysis, and remediation of security threats |
| Access Control | Identity management, authentication, and authorization |
| Vulnerability Management | Scanning, assessment, and remediation of vulnerabilities |
| Security Awareness | Staff training and phishing simulation programs |

## Key Principles

- **Defense in depth** — multiple layers of security controls
- **Least privilege** — minimum access necessary for roles
- **Zero trust** — verify every request, never trust by default
- **Continuous monitoring** — real-time threat detection and alerting
